# health-exporter

A lightweight HTTP health checker that exposes service status as Prometheus metrics.

Periodically probes a list of URLs and serves the results at `/metrics` in Prometheus text format.

```
service_up{url="https://www.google.com"} 1
service_up{url="https://my-api.internal"} 0
```

---

## How It Works

```
services.yaml ──▶ checker (worker pool, retry) ──▶ cache ──▶ GET /metrics ──▶ Prometheus
                       every 30s                   mutex
```

- Checks all URLs **concurrently** (5 workers)
- Each request has a **2s timeout** and **3 retries** before marking DOWN
- Results are cached — Prometheus gets instant response without waiting for a new check
- Shuts down cleanly on `Ctrl+C` / `SIGTERM`

## Project Structure

```
health-exporter/
├── cmd/exporter/main.go          # Entry point, HTTP server, graceful shutdown
├── internal/
│   ├── checker/engine.go         # Worker pool, HTTP probes, retry logic
│   ├── metrics/collector.go      # Thread-safe result cache
│   ├── models/config.go          # Config structs
│   └── scanner/scanner.go        # Auto-discover HTTP services on localhost
├── services.yaml                 # Target URLs (can be auto-generated with --scan)

```

---

## Flags

| Flag | Description |
|---|---|
| _(none)_ | Start the exporter using `services.yaml` |
| `--scan` | Scan localhost for HTTP services, write results to `services.yaml`, then exit |

## Quick Start

```bash
# 1. Install dependencies
go mod tidy

# 2. Build
go build -o health-exporter ./cmd/exporter

# 3. (Option A) Edit services.yaml manually, then run
./health-exporter

# 3. (Option B) Auto-detect services running on this machine first
./health-exporter --scan
./health-exporter

# 4. Verify
curl http://localhost:9090/metrics
```

---

## services.yaml

Auto-generated by `--scan`, or edit manually:

```yaml
services:
  - name: "My API"
    url: "http://localhost:8080"
  - name: "Google"
    url: "https://www.google.com"
```

### Ports scanned by `--scan`

`80, 443, 3000, 4000, 5000, 8000, 8080, 8081, 8443, 8888, 9000, 9090, 9091, 9092, 9093`

Tries `http://` first, falls back to `https://`. Any port that returns a response (any HTTP status code) is included.

## Deploy on a Server

Run all commands directly on the server console.

### Step 1 — Install Go

```bash
# Debian / Ubuntu
sudo apt update && sudo apt install -y golang-go

# RHEL / Rocky / AlmaLinux
sudo dnf install -y golang

# Verify
go version
```

### Step 2 — Clone and build

```bash
git clone https://github.com/internPholx/health-exporter.git
cd health-exporter
go build -o health-exporter ./cmd/exporter
```

### Step 3 — Scan for running services

```bash
./health-exporter --scan
```

Example output:

```
กำลังสแกน localhost...
พบ 3 services:
  - localhost:80   (http://localhost:80)
  - localhost:3000 (http://localhost:3000)
  - localhost:8080 (http://localhost:8080)
บันทึกลง services.yaml เรียบร้อย
```

> Skip this step if you prefer to edit `services.yaml` manually.

### Step 4 — Set up as a system service (systemd)

> For systems that use systemd (most modern Linux distributions).

```bash
sudo nano /etc/systemd/system/health-exporter.service
```

Paste:

```ini
[Unit]
Description=Health Exporter
After=network.target

[Service]
Type=simple
WorkingDirectory=/opt/health-exporter
ExecStart=/opt/health-exporter/health-exporter
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

Move the binary to the path defined above, then enable and start:

```bash
sudo mkdir -p /opt/health-exporter
sudo cp health-exporter services.yaml /opt/health-exporter/

sudo systemctl daemon-reload
sudo systemctl enable health-exporter
sudo systemctl start health-exporter
```

Verify it's running:

```bash
sudo systemctl status health-exporter
curl http://localhost:9090/metrics
```

View live logs:

```bash
sudo journalctl -u health-exporter -f
```

### Step 5 — Point Prometheus at the exporter

Add to your `prometheus.yml`:

```yaml
scrape_configs:
  - job_name: "health-exporter"
    static_configs:
      - targets: ["<SERVER_IP>:9090"]
```

Reload Prometheus:

```bash
sudo systemctl reload prometheus
```

Then open Prometheus UI → query `service_up` to see results.

## Re-scan after adding new services

When new services are deployed on the server:

```bash
/opt/health-exporter/health-exporter --scan
sudo systemctl restart health-exporter
```
